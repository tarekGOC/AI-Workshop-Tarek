# Workshop Project Requirements

This folder contains detailed requirement documents for all available workshop projects. Each project includes comprehensive user stories, acceptance criteria, system constraints, and technical specifications.

## Project Structure

Projects are organized by difficulty level:
- **00 - easy**: Beginner-friendly projects with simpler requirements
- **01 - medium**: Intermediate projects with multiple features
- **02 - hard**: Advanced enterprise-grade applications

## Project Difficulty Levels

### 🟢 Easy (Beginner-Friendly)
Perfect for getting started with AI-assisted development. These projects have simpler requirements and fewer user stories.

**Available Projects:**
1. **Todo List Application** - Simple task management with CRUD operations
2. **Temperature Converter** - Unit conversion tool with validation
3. **Password Strength Checker** - Security tool for password validation
4. **Expense Tracker (Simple)** - Basic personal finance tracker
5. **Quote of the Day App** - Random quote display application

### 🟡 Medium (Intermediate)
More complex applications with multiple user roles, authentication, and advanced features.

**Available Projects:**
1. **Inventory Management System** - Stock tracking and reporting
2. **URL Shortener** - Link shortening service with analytics
3. **Movie Rating Platform** - Community-driven movie reviews
4. **Appointment Booking System** - Scheduling and calendar management
5. **Document Management System** - File storage and collaboration
6. **Real Estate Platform** - Property listing and search

### 🔴 Hard (Advanced)
Enterprise-grade applications with complex architectures, real-time features, compliance requirements, and scalability considerations.

**Available Projects:**
1. **Real-time Collaborative Code Editor** - Multi-user code editing platform
2. **SaaS Project Management Platform** - Team collaboration tool
3. **E-commerce Platform** - Full-featured online store
4. **IoT Platform** - Device management and data processing
5. **Learning Management System (LMS)** - Educational content delivery
6. **Healthcare EHR System** - Electronic health records with compliance
7. **Microservices Platform** - Distributed architecture showcase

## How to Use These Requirements

1. **Choose Your Project**: Select one project that matches your skill level and interests
2. **Read the Full Document**: Open the markdown file for detailed specifications
3. **Review User Stories**: Understand all features and acceptance criteria
4. **Check Constraints**: Note system constraints and technical considerations
5. **Plan Your Implementation**: Map out your architecture and approach

## Universal Requirements

Regardless of difficulty level, **ALL projects must meet**:

### ✅ Functional Requirements
- **All acceptance criteria met** - Every checkbox in the requirement must be implemented
- **Complete functionality** - Full end-to-end working application

### ✅ Containerization
- **Docker containers** - All components packaged as containers
- **docker-compose.yml** - Single command deployment
- **Proper configuration** - Environment variables and networking

### ✅ Quality Standards
- **Testing** - Unit tests, integration tests per acceptance criteria
- **Security** - Input validation, SQL injection prevention, XSS protection
- **Error handling** - Graceful error handling throughout
- **Code quality** - Clean, maintainable code

### ✅ Documentation
- **README** - Setup, run, and test instructions
- **API documentation** - Endpoints and contracts (if applicable)
- **Database schema** - Data models (if applicable)

## Important Notes

⚠️ **All Acceptance Criteria Are Mandatory** - Every checkbox in your chosen requirement must be satisfied. Don't skip security, testing, or logging requirements.

⚠️ **Build From Scratch** - No templates or starter kits allowed. Demonstrate AI-assisted development skills.

⚠️ **Test Coverage Matters** - Most projects require minimum test coverage percentages and specific test types.
